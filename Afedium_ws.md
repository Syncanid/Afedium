# Afedium WebSocket 通信协议文档

本协议文档分为两部分：

1. **WS 服务器协议部分**：描述服务发现与认证流程。
2. **Afedium 协议部分**：定义控制码及客户端交互格式。

本协议面向客户端开发者与服务端维护人员，适用于自定义客户端实现、调试与集成。

---

## 📡 Part 1：WS服务器协议部分

### 1.1 服务发现机制

Afedium 服务端会周期性发送 UDP 广播，客户端通过监听即可发现本地局域网内的服务。

* **广播地址**：`255.255.255.255`
* **广播端口**：默认 `12840`（可配置）
* **广播内容**：JSON 格式字符串

#### 广播字段说明

| 字段名            | 类型      | 必须 | 描述                          |
|----------------|---------|----|-----------------------------|
| `server_name`  | String  | 是  | 服务名称（用户可见）                  |
| `service`      | String  | 是  | 服务标识，固定为 `"afedium_server"` |
| `ip`           | String  | 是  | 服务端 IPv4 地址                 |
| `port`         | Integer | 是  | WebSocket 端口号               |
| `UUID`         | String  | 是  | 唯一服务标识                      |
| `features`     | Object  | 是  | 支持功能清单（详见附录A）               |
| `using_auth`   | Boolean | 否  | 是否启用认证                      |
| `auth_timeout` | Integer | 否  | 认证超时秒数                      |
| `timestamp`    | String  | 否  | 发送时间（ISO 8601）              |

#### 广播示例

```json
{
  "server_name": "afedium_ws",
  "service": "afedium_server",
  "ip": "192.168.1.100",
  "port": 11840,
  "UUID": "a1b2c3d4-e5f6-7890-1234-567890abcdef",
  "features": {
    "info_access": true,
    "file_transfer": true,
    "file_management": true,
    "terminal": true,
    "system_upgrade": true
  },
  "using_auth": true,
  "auth_timeout": 60,
  "timestamp": "2025-06-30T10:00:00.123456"
}
```

### 1.2 认证协议

Afedium 使用自定义认证协议，采用问询-响应机制：

#### 流程

1. 客户端建立 WebSocket 连接
2. 服务端发送认证请求（控制码 `0x90`，带提示文本）
3. 客户端接收提示，获取用户输入
4. 客户端发送响应（控制码 `0x90`，内容为用户输入）
5. 服务端验证结果，并返回状态码：

   * `0x91`：认证成功
   * `0x92`：认证失败
   * `0x93`：认证超时

#### 控制码一览

| 控制码    | 方向        | 描述          |
|--------|-----------|-------------|
| `0x90` | 双向        | 认证请求 / 用户响应 |
| `0x91` | 服务端 → 客户端 | 认证成功        |
| `0x92` | 服务端 → 客户端 | 认证失败        |
| `0x93` | 服务端 → 客户端 | 认证超时        |

## ⚙️ Part 2：Afedium 协议部分

所有通信以二进制消息为主，首字节为控制码，其余为载荷。

### 2.1 必选指令

#### `0x03` 获取系统信息（必须）

* **客户端 → 服务端**： `chr(0x03) + <4字节UID>`
* **服务端 → 客户端**： `chr(0x03) + <UID> + <JSON对象>`
* JSON 可带格式化前缀（见附录B）

### 2.2 可选控制码

以下控制码是否支持取决于 UDP 广播中的 `features` 字段。

#### `0x01` 执行预定义命令

* 请求：`\x01 + UID + <命令字符串>`
* 响应：`\x01 + UID + <执行输出>`

#### `0x02` 通用信息访问（Get/Set）

* **只允许访问系统白名单（`static`, `dynamic`, `loaded_plugins`）下的节点。**
* 请求（Get）：`\x02 + UID + "get" + 变量名`
* 请求（Set）：`\x02 + UID + "set" + 变量名 + " " + 值`
* 响应：`\x02 + UID + <值或结果>`

#### `0x11` 文件上传（握手或小文件）

* 大文件握手：`\x11 + <路径> + 分块数 + SHA256`
* 小文件上传：`\x11 + <路径> + Base64数据 + 校验和`
* 响应：`\x11 + <会话ID 或 状态>`

#### `0x13` 文件下载请求

* 请求：`\x13 + <路径>`
* 响应：

  * 小文件：`\x13 + 编码类型 + Base64 + SHA256`
  * 大文件：`\x13 + 会话ID + 分块数 + SHA256`

#### `0x1B` 确认开始下载

* 请求：`\x1B + <会话ID>`
* 触发：服务端使用 `0x18` 开始发送分块

#### `0x18` 文件分块传输

* 客户端上传：`\x18 + 会话ID + 块索引 + Base64`
* 服务端下载：`\x18 + 会话ID + 块索引 + Base64`
* 结束标志：`\x18 + END`

#### `0x19` 请求重传

* 请求：`\x19 + 会话ID + JSON数组[丢失索引]`

#### `0x1A` 取消下载

* 请求：`\x1A + 会话ID`
* 响应：`\x1A + CANCELLED`

#### 文件系统操作指令

| 控制码    | 功能       | 请求格式示例                   |
|--------|----------|--------------------------|
| `0x12` | 删除文件/目录  | `\x12 + /path/to/file`   |
| `0x14` | 列出目录内容   | `\x14 + /path`           |
| `0x15` | 获取状态信息   | `\x15 + /file.txt`       |
| `0x16` | 创建目录     | `\x16 + /new/folder`     |
| `0x17` | 重命名文件/目录 | `\x17 + old.txt new.txt` |

响应统一为相同控制码 + 结果内容

## 📌 附录 A：功能标识（features）

Afedium模块可额外定义功能标识，不以表格为准

| 标识符               | 类型      | 描述               | 控制码                  |
|-------------------|---------|------------------|----------------------|
| `info_access`     | Boolean | 支持读写服务端变量        | `0x02`               |
| `file_transfer`   | Boolean | 支持上传下载文件         | `0x11` `0x13` 等      |
| `file_management` | Boolean | 支持删除/重命名/列出等文件操作 | `0x12` \~ `0x17`     |
| `terminal`        | Boolean | 支持终端指令执行         | `0x01`               |
| `system_upgrade`  | Boolean | 支持系统热更新指令        | `0x01: core upgrade` |

## 📌 附录 B：`0x03` 格式化协议规范

Afedium 支持带前缀的 JSON 键名以控制 UI 渲染样式。

#### B.1 前缀规则（开头以 `/`，单字符标记）

| 前缀       | 描述                                                          |
|:---------|:------------------------------------------------------------|
| **(无)**  | **默认 (Default):** 以标准的“标签: 值”格式显示。                          |
| **`/T`** | **主标题 (Title):** 作为页面的主标题或卡片的核心标题，使用大号、加粗字体。                |
| **`/S`** | **副标题 (Subtitle):** 用于信息分组的次级标题，字体比正文稍大或有特殊样式。              |
| **`/B`** | **加粗 (Bold):** 值的文本部分以加粗样式显示。                               |
| **`/I`** | **斜体 (Italic):** 值的文本部分以斜体样式显示。                             |
| **`/P`** | **状态点 (Point):** 在Key的文本之后显示一个基于布尔值的彩色状态点或小型图标。             |
| **`/K`** | **状态指示器 (Key Status):** 用于单个布尔值。将根据布尔值在键旁边显示不同颜色和形状的图标。     |
| **`/C`** | **状态芯片组 (Chip Group):** 仅用于值为Map类型。将Map中的每个键值对渲染成一个独立的状态芯片。 |
| **`/L`** | **链接 (Link):** 将值作为URL，显示为可点击的超链接。                          |
| **`/A`** | **追加 (Append):** 将此键值对追加到前一个非追加项目的末尾，形成一个水平排列的组。            |
| **`/H`** | **隐藏 (Hidden):** 客户端不应在UI上显示此键值对，用于传递元数据。                   |
| **`/V`** | **仅显示值 (Value-only):** 客户端应隐藏此键的标签，仅在UI上显示其值。               |

* 多个前缀以多个 `/` 连写，例如：`/P/B/状态名`
* 使用 `//` 转义真实 `/` 字符
